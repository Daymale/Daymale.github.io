Veröffentlicht am 2017-02-11

免费搭建个人博客-hexo+github

使用hexo生成静态博客并架设在免费的github page平台

准备

系统

  • window 10 64位

使用软件

  • Git v2.11.164位 下载地址：百度云
  • Node.js v6.9.5 64位 下载地址：百度云

安装Git

  • 大部分都默认安装，一路点击”Next”

  • 这里选择对应的windows选项

  • 等待安装完成

安装Node.js

  • Node.js安装更加简单，还是一路”Next”,最后等待安装完成即可


    验证软件正确安装

  • 点击开始菜单>运行，输入”cmd”

    然后输入以下命令，结果与下图相同则说明安装正确，若不正确可卸载从新安装

申请GitHub账号

  • 点击 GitHub进入官网注册账号

  • 分别输入用户名、邮箱、密码，点击注册

  • 按默认设置，点击”Finish sign up”

  • 点击”New repositorys”,新建一个版本库

  • 输入Repository name:++Daymale.github.io++

    (Daymale与你注册的用户名一致，这个就是你博客的域名了)

    GitHub账号申请完成

  • 启用GitHub page

    点击右边的”Setting”菜单进入设置，点击”Launch antomatic page generator”

    点击底部的”Continue to layouts”

    最后点击”Publish page”,发布GitHub默认生成的一个静态站点

  • 然后右键单击选择”Git Bash”

    按住Ctrl+C即可停止

管理博客

发布博客

配置SSH keys

我们如何让本地git项目与远程的GitHub建立联系呢？用SSH keys

  • 生成新的SSH key：

注意1：此处的邮箱地址，你可以输入自己的邮箱地址

注意2：此处的-C是大写的C

然后系统会要求你输入密码：

在回车中会提示你输入一个密码，这个密码会在你提交项目是使用，如果为空的话提交项目则不用输入。这个设置是防止别人往你的项目里提交内容。

注意：输入密码的时候是没有 * 字样的，你直接输入就可以了。

最后看到这样的界面，就成功设置SSHkey了：

  • 添加SSH key到GitHub

    在本机设置SSH key之后，需要添加到GitHub上，以完成SSH链接的设置。

    1、打开本地C:\Users\Administrator\ .ssh\id_rsa.pub文件。此文件里面内容为刚才生成人密钥。如果看不到这个文件，你需要设置显示隐藏文件。准确的复制这个文件的内容，才能保证设置的成功。

    2、登陆github系统。点击右上角的 Account Settings—>SSH Public keys —> add another public keys

    3、把你本地生成的密钥复制到里面（key文本框中）， 点击 add key 就ok了

hexo教程

Veröffentlicht am 2017-02-11

hexo新建一篇文章给它添加分类和标签

输入命令

通过编辑器打开：添加categories和tags选项

---
title: hexo教程 # 文章标题
date: 2017-02-11 23:47:44 # 文章生成时间
categories: # 文章分类目标，可以忽略
tags: # 文章标题 可以忽略
    - 标签1
    - 标签2
description: # 你对本页的描述 可以忽略
---

menu-自定义页面及菜单

  • 添加菜单
    更改主题配置文件(blog\themes\maupassant\ _config.yml)

    menu:

    - page: home
  directory: .
  icon: fa-home
- page: archive
  directory: archives/
  icon: fa-archive
- page: about
  directory: about/
  icon: fa-user
- page: rss
  directory: atom.xml
  icon: fa-rss

按照上面已有格式填写，填写后请在source目录下建立相应名称的文件夹，并包含index.md文件，以正确显示页面。

导航菜单中集成了FontAwesome图标字体，可以在这里选择新的图标，并按照相关说明使用。

  • 支持语言

    更改站点配置文件(blog\ _config.yml)下的language: #语言代码

    博客语言

    language: zh-CN
    目前支持简体中文（zh-CN），繁体中文（zh-TW），英语（en），法语（fr-FR），德语（de-DE），韩语（ko）和西班牙语（es-ES），欢迎翻译至其它语言。

开启评论

  • 我使用多说代替自带的评论，在多说 网站注册 > 后台管理 > 添加新站点 > 工具 === 复制通用代码 里面有 short_name
  • 在根目录 _config.yml 添加一行 disqus_shortname: jslite 是在多说注册时产生的
  • 复制到 themes\landscape\layout_partial\article.ejs

常用命令

hexo

>
npm install hexo -g #安装
npm update hexo -g #升级
hexo init #初始化

简写

>
hexo n “我的博客” == hexo new “我的博客” #新建文章
hexo p == hexo publish
hexo g == hexo generate#生成
hexo s == hexo server #启动服务预览
hexo d == hexo deploy#部署

服务器

>
hexo server #Hexo 会监视文件变动并自动更新，您无须重启服务器。
hexo server -s #静态模式
hexo server -p 5000 #更改端口
hexo server -i 192.168.1.1 #自定义 IP

hexo clean #清除缓存 网页正常情况下可以忽略此条命令
hexo g #生成静态网页
hexo d #开始部署

监视文件变动

>
hexo generate #使用 Hexo 生成静态文件快速而且简单
hexo generate –watch #监视文件变动

完成后部署

>
两个命令的作用是相同的
hexo generate –deploy
hexo deploy –generate

hexo deploy -g
hexo server -g
